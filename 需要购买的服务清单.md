# 需要购买的外部服务清单

## 概述

本项目目前部分功能使用 Mock（模拟）模式实现，上线生产环境前需要接入真实的外部服务。以下是需要购买和配置的服务清单。

---

## 一、短信验证码服务

### 当前状态
- ✅ 已实现 Mock 模式：生成6位验证码存入 Redis，直接返回给前端
- ❌ 未接入真实短信网关

### 业务场景
- 用户注册时发送验证码
- 登录时发送验证码（可选）
- 找回密码时发送验证码（可选）

### 需要的服务
| 服务类型 | 推荐服务商 | 费用参考 |
|---------|-----------|---------|
| 短信验证码 | 阿里云短信服务（Dysmsapi） | 约 0.045 元/条 |
| 短信验证码 | 腾讯云短信服务 | 约 0.035 元/条 |
| 短信验证码 | 极光短信 | 约 0.03 元/条 |

### 配置需求
```yaml
# application.yml 需要添加
app:
  sms:
    access-key-id: # 服务商AK
    access-key-secret: # 服务商SK
    sign-name: # 短信签名（需审核）
    template-code: # 验证码模板（需审核）
```

### 修改文件
- `backend/src/main/java/com/example/catguardian/service/impl/AuthServiceImpl.java`
  - 修改 `sendVerificationCode` 方法，调用真实短信 API

### 需要的服务
| 服务类型 | 推荐服务商 | 费用参考 |

| 邮件推送服务 | 腾讯云邮件服务 | 按需付费 |

### 配置需求
```yaml
# application.yml 需要添加

  mail:
    host: # SMTP服务器地址
    port: # SMTP端口
    username: # 邮箱账号
    password: # 邮箱密码/授权码
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
```

## 四、支付服务

### 当前状态
- ✅ 已实现订单状态管理（创建→接单→服务中→完成→评价）
- ✅ 已实现退款规则（待接单全额退、已接单扣5%、服务当天扣10%）
- ❌ 未接入真实支付渠道（微信支付、支付宝）

### 业务场景
- 上门喂猫订单支付
- 商城商品购买支付
- 积分兑换支付
- 订单退款

### 需要的服务
| 服务类型 | 推荐服务商 | 费用参考 |
|---------|-----------|---------|
| 微信支付 | 微信商户平台 | 费率 0.6% |
| 支付宝 | 支付宝商户平台 | 费率 0.6% |
| 聚合支付 | 支付宝/微信官方 | 无额外费用 |

### 认证要求
- **必须完成企业认证**（个体工商户或企业营业执照）
- 需要开通微信商户号/支付宝商户号
- 需要配置商户密钥、API证书

### 配置需求
```yaml
# application.yml 需要添加
payment:
  wechat:
    app-id: # 微信小程序AppID
    merchant-id: # 微信商户号
    api-key: # 微信API密钥
    cert-path: # API证书路径
  alipay:
    app-id: # 支付宝AppID
    private-key: # 私钥
    public-key: # 公钥
    gateway-url: # 支付宝网关
```

### 需要新增的文件
- `backend/src/main/java/com/example/catguardian/service/PaymentService.java` - 支付服务接口
- `backend/src/main/java/com/example/catguardian/service/impl/PaymentServiceImpl.java` - 支付服务实现
- `backend/src/main/java/com/example/catguardian/controller/PaymentController.java` - 支付回调控制器

### 修改文件
- `backend/src/main/java/com/example/catguardian/service/impl/OrderServiceImpl.java`
  - 修改支付相关逻辑，调用真实支付 API
